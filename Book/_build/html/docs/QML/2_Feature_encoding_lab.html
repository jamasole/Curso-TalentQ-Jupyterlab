2. Feature encoding#

2.1. Introducci√≥n#

El proceso de feature encoding se encarga de codificar datos cl√°sicos en estados cu√°nticos. El problema de codificar datos cl√°sicos en estados cu√°nticos es esencial en todas las disciplinas que estudian sacar ventaja de las propiedades cu√°nticas para problemas cl√°sicos, como la comunicaci√≥n, criptograf√≠a o la computaci√≥n cu√°ntica. Este notebook se centra en las t√©cnicas de feature encoding b√°sicas para QML. Se empieza con un conjunto de datos \(\mathbf{x}\) Conjunto que se quiere codificar en un estado \(\ket{\psi_x}\) con el que trabajar, para ello se construye un circuito parametrizado que generar√° una unitaria \(\mathcal{U}(\mathbf{x})\) tal que \(\ket{\psi_x}=\mathcal{U}(\mathbf{x})\ket{0}\). Dependiendo del tipo de feature encoding que se utilice, la unitaria \(\mathcal{U}(x)\) tendr√° diferentes caracter√≠sticas. Se trata de un proceso crucial para el dise√±o y aplicaci√≥n de los algoritmos, ya que influye en su capacidad computacional y en su eficiencia [1][2][3][4].

Se define un conjunto de datos de entrada \(D\) de dimensi√≥n \(M\) x \(N\), es decir, se cuenta con un dataset con M instancias y N variables. Este dataset se puede expresar como \(D = \{\mathbf{x}_{(1)}, ..., \mathbf{x}_{(m)}, ..., \mathbf{x}_{(M)}\}\) donde cada \(\mathbf{x}_{(m)}\) es un vector de dimensi√≥n N.

Hay multiples t√©cnicas de feature encoding, algunas se detallan a continuaci√≥n. Como ejemplo intentaremos codificar el vector \(x=(0,1,2,3)\) de diferentes maneras en nuestro circuito.

2.2. Tipos de feature encoding#

2.2.1. Basis encoding#

La t√©cnica Basis encoding, tambi√©n conocida como Basis Embedding, asocia un input (entrada) a un estado de la base computacional de un sistema de qubits. Es por eso que los datos cl√°sicos deben estar, o poder ser representados, por una cadena de bits, es decir, mediante este m√©todo de codificaci√≥n se trabajar√° con cadenas binarias. Este m√©todo representa el dato cl√°sico \(x=1101\) como el estado \(\ket{1101}\), por ejemplo.

Se considera el conjunto de datos \(D\), definido anteriormente. En el caso de basis encoding, las N caracter√≠sticas de cada ejemplo deben estar representadas por cadenas binarias, es decir, \(\mathbf{x}_{m} = (b_1,...,b_N)\) con \(b_i\in \{ 0,1\}\) para \(i=1,...,N\). Asumiendo que todas las variables (caracter√≠sticas?) del dataset se representan mediante un √∫nico bit, cada input (entrada) \(\mathbf{x}_{m}\) puede representarse como \(\ket{\mathbf{x}_{m}}\). Extendiendo la expresi√≥n anterior a todo el dataset, obtenemos la representaci√≥n del conjunto de datos al completo, √©sta corresponde a la siguiente expresi√≥n:

\[\ket{D} = \frac{1}{\sqrt M} \sum^M_{m=1} \ket{\mathbf{x}_{m}}\]

Por ejemplo, si consideramos que el dataset x los datos en binario correspondria a ejemplos \(x = 1111011\), la representaci√≥n de los mismos se corresponde con \(\ket{1111011}\). En este caso, el estado corresponde a:

\[\ket D = \ket{1111011}\]
num_q=7

def Basis_encoding(x, nqubits= 2):
    c= Circuit(nqubits=nqubits)
    
    for i in range(nqubits):
        if x[i]==1:
            c.add(gates.X(q=i))
    return c

x = np.array([1,1,1,1,0,1,1])
c_b=Basis_encoding(x,num_q)


print("x               : ", x)
print("amplitude vector: ", np.array(c_b.execute().state()))
print(c_b.draw())

2.2.2. Amplitude encoding#

La t√©cnica Amplitude encoding o amplitude embedding, codifica los datos cl√°sicos como las amplitudes de un estado cu√°ntico. En este caso, una instancia del dataset que cl√°sicamente se representa mediante un vector normalizado (denominado \(x\)) de dimensi√≥n N , aplicando esta t√©cnica de codificaci√≥n, dicho vector corresponde a las amplitudes del estado cu√°ntico de n-qubits. La expresi√≥n es la siguiente:

\[ \ket{\psi_x} = \sum^N_{i=1} x_i\ket i\]

Donde \(N=2^n\), \(x_i\) es el i-√©simo elemento del vector \(\mathbf{x}\) y \(\ket i\) es el i-√©simo estado de la base computacional. A diferencia de la t√©cnica, basis encoding, \(x_i\) puede tomar valores de distintos tipos, como integer (valores enteros) o float (valores de coma flotante).

Como ejemplo, si tratamos de codificar la instancia correspondiente al vector utilizando esta t√©cnica obtendremos:

\[\ket{\psi_x} = \frac{1}{4} [\ket{01}+2\ket{10}+3\ket{11}]\]

Nota: Todos los vectores cu√°nticos deben estar normalizados, \(\bra{\psi_x}\ket{\psi_x} = 1\)

num_q=2

def Normalize(x):
    N=np.linalg.norm(x)
    return 1/N*x

x = np.array([0,1,2,3])
#We normalize x
x_norm=Normalize(x)
c_a = Circuit(num_q)

print("x               : ", x)
print("amplitude vector: ", np.array(c_a(x_norm,nshots=10000).state()))
print(c_a.draw())

2.2.3. Angle encoding#

Angle encoding es una t√©cnica de codificaci√≥n b√°sica, las N caracter√≠sticas de las instancias del dataset se codifican como √°ngulos de rotaci√≥n de N c√∫bits. Esta metodolog√≠a codifica N caracter√≠sticas en los √°ngulos de rotaci√≥n de N qubits. Dichas rotaciones pueden llevarse a cabo en cualquier eje, tanto en el X, como en el Y o en el Z.

En general, para codificar una instancia \(\mathbf{x}=(x_1,...,x_N)\) se utiliza la siguiente expresi√≥n:

\[ \ket {\mathbf{x}} = \bigotimes^N_{i=1} \cos(x_i)\ket 0 + \sin(x_i)\ket 1 \]

Este m√©todo es diferente a los dos m√©todos anteriores, ya que √∫nicamente codifica una instancia cada vez en lugar de trabajar con todo el dataset. No obstante, utiliza N qubits y la profundidad del circuito es constante, por lo que resulta compatible con el hardware cu√°ntico.

Esta t√©cnica se puede representar mediante una matriz unitaria:

\[ S_{\mathbf{x}_j} = \bigotimes^N_{i=1} U(\mathbf{x}_{j_i}) \]

donde:

\[\begin{split} U(\mathbf{x}_{j_i}) = \begin{bmatrix} \cos(\mathbf{x}_{j_i}) & -\sin (\mathbf{x}_{j_i}) \\ \sin (\mathbf{x}_{j_i}) & \cos (\mathbf{x}_{j_i}) \end{bmatrix} \end{split}\]

Por ejemplo, si se trata de codificar el vector\(\mathbf{x}=(0,1,2,3)\) con este tipo de encoding se necesitar√°n cuatro c√∫bits y quedar√° como sigue: $\( \ket {\mathbf{x}} = (\cos(0)\ket 0 + \sin(0)\ket 1 ) \otimes (\cos(1)\ket 0 + \sin(1)\ket 1) \otimes (\cos(2)\ket 0 + \sin(2)\ket 1) \otimes (\cos(3)\ket 0 + \sin(3)\ket 1 ) \)$

num_q=4

def Angle(x,nqubits):
    c= Circuit(nqubits=nqubits)
    
    for i in range(nqubits):
        c.add(gates.RX(q=i,theta=x[i]))
    return c

x = np.array([0,1,2,3])

c_an=Angle(x,num_q)

2.2.4. Dense angle encoding#

Esta t√©cnica es una generalizaci√≥n de la codificaci√≥n anterior, es capaz de codificar dos caracter√≠sticas por cada qubit, haciendo uso de fases relativas. En este caso, la instancia \(x=(x_1,...,x_N)\) se codifica como sigue:

\[ \ket x = \bigotimes^{N/2}_{i=1} \cos(x_{2i-1})\ket 0 + e^{ix_{2i}}\sin(x_{2i-1})\ket 1 \]

Aunque dense angle encoding y angle encoding utilizan funciones sinusoidales y exponenciales se pueden sustituir por funciones unitarias arbitrarias para generar otras t√©cnicas de feature encoding.

Por ejemplo, si se trata de codificar el vector\(\mathbf{x}=(0,1,2,3)\) con este tipo de encoding se necesitar√°n dos c√∫bits y quedar√° como sigue: $\( \ket {\mathbf{x}} = (\cos(0)\ket 0 + e^{i1}\sin(0)\ket 1 ) \otimes (\cos(2)\ket 0 + e^{i3}\sin(2)\ket 1) \)$

num_q=2

def denseAngle(x,nqubits):
    c= Circuit(nqubits=nqubits)
    
    for i in range(nqubits):
        c.add(gates.RX(q=i,theta=x[i]))
        c.add(gates.RZ(q=i,theta=x[i+1]))
    return c

x = np.array([0,1,2,3])

c_dan=denseAngle(x,num_q)

print("x               : ", x)
print("amplitude vector: ", np.array(c_dan.execute().state()))

2.3. C√≥mo escoger Feature Encoding#

Cuando se trata con circuitos variacionales la decisi√≥n de qu√© feature encoding utilizar es crucial. Los diferentes feature encoding presentan diferentes ventajas e inconvenientes dependiendo del problema que queramos resolver. Las t√©cnicas mencionadas en este notebook se pueden separar en dos tipos:

  • basis encoding donde se trabaja con los elementos de la base computacional como inputs (entradas)

  • el resto de codificadores que trabajan con las amplitudes del vector cu√°ntico.

La t√©cnica de basis encoding presenta la capacidad de calcular operaciones no lineales de forma natural, a cambio es el codificador que presenta mayor n√∫mero de problemas:

  • Es el m√°s susceptible a errores.

  • Escala muy mal con el n√∫mero de c√∫bits, ya que de esto depende la precisi√≥n de los feature que se quiere codificar.

  • Es el m√©todo que presenta mayor dificultad a la hora de entrenar la red neuronal y tiene tendencias a presentar barren plateaus (gradientes que tienden a cero).

El resto de m√©todos basados en codificar en las amplitudes del vector cu√°ntico no presentan los inconvenientes enumerados anteriormente y son los m√©todos preferidos para desarollar algoritmos en la era del NISQ (Noisy intermediate-scale quantum). A√∫n as√≠, presentan el inconveniente que hay que escoger sabiamente c√≥mo introducir la no-linealidad necesaria para cada problema. La forma m√°s gen√©rica es utilizar Amplitude encoding y aplicar una funci√≥n \(f(x)\) a tus datos para obtener correlaciones entre ellos y as√≠ poder resolver problemas no-lineales. Este m√©todo, que es a primera vista sencillo, presenta el problema que la amplitud del circuito puede ser muy grande dependiendo de \(f(x)\) y adem√°s, escoger \(f(x)\) puede ser un proceso muy arbitrario. Por otra parte, otros m√©todos de codificaci√≥n m√°s sencillos pueden proporcionar directamente los elementos de no linealidad necesarios para resolver el problema.

Escoger el m√©todo de feature encoding es un proceso crucial para un VQC (Variational Quantum Circuit) pero, como se ha visto, no es una ci√©ncia exacta, por lo que se deben probar distintas t√©cnicas y escoger la que mejor se ajusta al problema en particular. Para finalizar, existen otras t√©cnicas m√°s sofisticadas para introducir datos cl√°sicos en un circuito variacional, una de ellas es conocido como data re-uploading [7].

ANEXO NOTACI√ìN

Para que la comprensi√≥n de los notebooks sea mejor se ha unificado la notaci√≥n utilizada en los mismos. Para diferenciar un vector de un valor √∫nico se har√° uso de la negrita. De manera que \(\mathbf{x}\) corresponde a un vector y \(z\) ser√° una variable de una √∫nica componente.

Si se quiere hacer referencia a dos vectores distintos pero que pertenecen al mismo dataset se utilizar√° un sub√≠ndice, es decir, \(\mathbf{x_i}\) har√° referencia al i-√©simo vector del dataset. Si se quiere referenciar una caracter√≠stica concreta del vector \(\mathbf{x_i}\) se a√±adir√° un nuevo sub√≠ndice, de manera que \(\mathbf{x_{i_j}}\) har√° referencia a la j-√©sima variable del i-√©simo vector.
