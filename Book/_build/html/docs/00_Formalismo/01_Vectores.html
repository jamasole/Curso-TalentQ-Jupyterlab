Espacio Vectorial#

Comenzaremos definiendo los vectores de una forma operativa, y despu√©s haremos hincapi√© en su car√°cter abstracto.

Definici√≥n: Vector

La forma operacional de definir un vector de dimensi√≥n \(N\) consiste en lista (columna) de \(N\) n√∫meros complejos

\[\begin{split} |u\rangle = \begin{pmatrix} {u_1}\\ {u_2}\\ \vdots \\ {u_N} \end{pmatrix} \end{split}\]

  • El s√≠mbolo \(\ket{u}\) representa al vector y se denomina ket en la notaci√≥n de Dirac. Se trata simplemente de una notaci√≥n alternativa a la usual, donde un vector se representa en la forma \({\bf u}\) o tambi√©n \(\vec u\)

  • Los n√∫meros complejos \(u_i \in {\mathbb C}\) con \(\, i=1,...,N\) se denominan componentes del vector \(\ket{u}\)

Definici√≥n: Espacio Vectorial

La colecci√≥n de todos los posibles vectores de \(N\) componentes, con las propiedades de suma y multiplicaci√≥n forman un espacio vectorial, \(\V\) de dimension compleja \(N\)

es decir, en un espacio vectorial tenemos dos operaciones posibles:

  • sumar dos vectores

\[\begin{split} |u\rangle + \ket{v}~ =~\, \begin{pmatrix} {u_1}+v_1\\ {u_2}+v_2\\ \vdots \\ {u_N}+v_n \end{pmatrix} ~= ~\ket{w} \end{split}\]

  • multiplicar un vector por n√∫mero complejo \(\lambda\in {\mathbb C}\)

\[\begin{split} \lambda|u\rangle ~ =~ \begin{pmatrix} {\lambda u_1}\\ {\lambda u_2}\\ \vdots \\ {\lambda u_N} \end{pmatrix} ~\equiv~\ket{\lambda u} \end{split}\]

  • todo vector de \(V\) se denota mediante el s√≠mbolo \(\ket{v}\) menos uno, el elemento neutro que se escribe como \(0\).

  • La existencia de un elemento opuesto y de un elemento neutro son dos propiedades importantes en un espacio vectorial

(4)#\[\begin{eqnarray} \ket{v} + 0 &=& \ket{v} \nonumber\\ \ket{v} + \ket{\hbox{-}v} &=& \ket{v}-\ket{v} = 0 \nonumber\\ \end{eqnarray}\]

Nota

La dimensi√≥n es igual al n√∫mero de cantidades (grados de libertad) que debemos fijar para especificar un vector. Pero \(N\) n√∫meros complejos equivalen a \(2N\) n√∫meros reales.

Entonces, podemos decir que: la dimensi√≥n compleja de un espacio vectorial complejo \(\V\) es \(N\), o que su dimensi√≥n real es \(2N\)

$\( {\rm dim}_{\mathbb C} \V = N ~~~~\Longleftrightarrow ~~~ {\rm dim}_{\mathbb R} \V = 2N \)$

Base#

Es evidente que el vector \(\ket{v}\) definido anteriormente se puede expandir de la siguiente manera


\[\begin{split} \ket{u} ~=~ \begin{pmatrix} {u_1} \\ {u_2} \\{u_3}\\ \vdots \\ \,{u_{N-1}}\, \\ {u_{N}} \end{pmatrix}~= =~ {u_1} \begin{pmatrix} 1 \\ 0 \\ 0\\ \vdots \\ 0 \\ 0 \end{pmatrix} \,+\,{u_2} \begin{pmatrix} 0 \\ 1 \\ 0\\ \vdots \\ 0 \\ 0 \end{pmatrix}~+~ ... ~+ ~ {u_{N-1}} \begin{pmatrix} 0 \\ 0 \\ 0\\\vdots \\ 1 \\ 0 \end{pmatrix}+ \,{u_N}\, \begin{pmatrix} 0 \\ 0 \\0\\ \vdots \\ 0 \\ 1 \end{pmatrix} \end{split}\]

Esta notaci√≥n se vuelve inmanejable cuando \(N\) se hace muy grande. Sin embargo, la notaci√≥n de Dirac equivalente es muy concisa. Definamos la siguiente colecci√≥n \(\{ e_i\}\) de vectores


\[\begin{split} |1\rangle \sim \begin{pmatrix} 1 \\ 0 \\ 0\\ \vdots \\ 0 \\ 0 \end{pmatrix}~~~~ ~~~|2\rangle \sim \begin{pmatrix} 0 \\ 1 \\ 0\\ \vdots \\ 0 \\ 0 \end{pmatrix}~~~~~~~~~ \cdots ~~~~~~~~ ~~|{N-1}\rangle \sim \begin{pmatrix} 0 \\ 0 \\ 0\\\vdots \\ 1 \\ 0 \end{pmatrix}~~~~ ~~|N\rangle \sim \begin{pmatrix} 0 \\ 0 \\0\\ \vdots \\ 0 \\ 1 \end{pmatrix} \end{split}\]

Entonces simplemente

\[ \ket{u} ~=~ {u_1} |1 \rangle + {u_2} | 2\rangle +... + {u_{ N}}|{ N}\rangle~=~ \sum_{i=1}^N {u_ i} |i\rangle \]

Lo √∫nico que hemos hecho es representar el vector \(\ket{u}\) en una base. Vamos a formalizar esto.

Definici√≥n: Base

En un espacio vectorial \(V\) de dimensi√≥n \(N\) una base es una colecci√≥n \(\{\ket{e_1},...,\ket{e_N}\}\) de \(N\) vectores linealmente independientes

Cualquier vector \(\ket{v}\in V\) que no sea el elemento neutro \(\ket{v}\neq 0\), se puede expresar como una combinaci√≥n lineal de los vectores de una base

\[ \ket{v} = \sum_{i=1}^N v_i \ket{e_i} ~=~ v_1\ket{e_1} \,+\, v_2\ket{e_2}\, +\, ...\, +\, v_N \ket{e_N} \]

Los coeficientes \(v_i\) son las componentes de \(\ket{v}\) en la base dada \( \{ \ket{e_i} \}\).

Nota

Los s√≠mbolos \(e_i\) son cualquier colecc√≥n de etiquetan que sirva para distinguir entre los \(N\) estados de la base. En el ejemplo anterior hemos tomado \(e_i = i \in (1,...,N)\).

Por ejemplo, con \(N=4\) podr√≠amos usar tambi√©n notaci√≥n binaria

\[ \ket{e_0} = \ket{0} = \ket{00}~~~~~~,~~~~~~\ket{e_1} = \ket{1} = \ket{01}~~~~~~,~~~~~~\ket{e_2} = \ket{2} = \ket{10}~~~~~~,~~~~~~\ket{e_3} = \ket{3} = \ket{11} \]

Podemos usar la clase Statevector de qiskit para tener una expansi√≥n en la base \(\{\ket{e_i}=\ket{b(i)}\}\), donde \(b(i)\) es la representaci√≥n binaria del √≠ndice \(i\). Por ejemplo \(b(3) = 11\).

Vectores: definici√≥n intr√≠nseca#

La definici√≥n de vector que hemos introducido antes est√° asociada a una base concreta, en la cu√°l, los n√∫meros \(v_i\) representan las componentes de dicho vector en esa base.

Sin embargo un vector es un objeto abstracto, que admite una representaci√≥n distinta en cada base. Para ellos debemos comenzar por hablar de las distintas bases.

Cambio de base

En un espacio vectorial de dimensi√≥n finita \(N\), existen infinitas bases . Todas ellas sirven para representar un vector arbitrario.

Consideremos dos bases \(\{\ket{e_i}\}\) y \(\{\ket{\tilde e_j}\}\) donde \( i,j = 1,...,N\). Por ser bases, un vector cualquier admitir√° una representaci√≥n en cada una de ellas

$\(~~\ket{v} = \sum_{i} \color{blue}{v_i} \color{blue}{\ket{e_i}} = \sum_j \color{red}{\tilde v_j} \color{magenta}{\ket{\tilde e_j}}\)$

Las listas de componentes \(v_i\) y \(\tilde v_i\) contienen la misma informaci√≥n, puesto que representan al mismo vector \(\ket{v}\), cada una en una base

Por tanto: debe ser posible entonces encontrar la traducci√≥n de una lista de valores a la otra.

Es evidente que este traductor debe depender de la relaci√≥n que guardan ambas bases \(\{\ket{e_i}\}\) y \(\{\ket{\tilde e_i}\}\) entre s√≠

Por ser bases, cualquier elemento (vector) de una se puede expresar como una combinaci√≥n lineal de elementos de la otra. Por ejemplo

\[ \ket{\tilde e_j} = \sum_{i=1}^N C_{ i j} \ket{e_i} ~~~~~~~j=1,..., N \]

Los coefficientes \(C_{ i j}\in {\mathbb C}\) constituyen la matriz de cambio de base

Notar que de los sub√≠ndices de \(C_{ij}\)

  • el √≠ndice \(i\) est√° sumado con los elementos de la base

  • el √≠ndice \(j\) es una etiqueta que recuerda que estamos expandiendo el vector \(\ket{\tilde e_j}\)

Ejemplo

Sea la matriz de coeficientes es

\[\begin{split} C_{ij} = \begin{pmatrix} 1 & 1 \\ i & -i \end{pmatrix} = \begin{pmatrix} C_{11} & C_{12} \\ C_{21} & C_{22} \end{pmatrix}\, , \end{split}\]

Entonces la expresi√≥n \(\ket{\tilde e_j} = \sum_{i=1}^N C_{ i j} \ket{e_i}\) en este caso relaciona las dos bases en la forma siguiente

(5)#\[\begin{eqnarray} \ket{\tilde e_1} &=& C_{11}\ket{e_1} + C_{21}\ket{e_2} = \ket{e_1} + i\ket{e_2}\nonumber\\ \rule{0mm}{8mm} \ket{\tilde e_2} &=& C_{12}\ket{e_1} + C_{22}\ket{e_2}=\ket{e_1} - i\ket{e_2} \nonumber \end{eqnarray}\]

El cambio de base, se puede escribir como una multiplicaci√≥n de matrices colocando los vectores en una fila

\[\begin{split} \big(\ket{\tilde e_1},\ket{\tilde e_2}\big) = \big(\ket{e_1},\ket{e_2}\big) \, \begin{pmatrix} 1 & 1 \\ i & -i \end{pmatrix} = \big(\ket{e_1} + i\ket{e_2}, \ket{e_1}-i\ket{e_2} \big) \end{split}\]

Nota

La forma en que est√°n sumados los √≠ndices

\[ \ket{\tilde e_j} = \sum_{\color{blue}{i=1}}^{\color{blue}{N}} C_{ \color{blue}{i} j} \ket{e_{\color{blue}{i}}} \]

Esto hace que, si queremos escribir esta ecuaci√≥n en notaci√≥n matricial, debamos poner los vectores en una fila

\[\begin{split} \begin{pmatrix} \ket{\tilde e_1} & \cdots & \ket{\tilde e_N} \end{pmatrix} = \begin{pmatrix} \ket{ e_{\color{blue}{1}}} & \cdots & \ket{e_ {\color{blue}{N}}}\end{pmatrix} \begin{pmatrix} C_{\color{blue}{1}1} & \cdots & C_{\color{blue}{1}N} \\ \vdots & \ddots & \vdots \\ C_{\color{blue}{N}1} & \cdots & C_{\color{blue}{N}N} \end{pmatrix} \end{split}\]

Ahora podemos obtener el diccionario entre componentes \(v_i\) y \(\tilde v_i\), a partir de la matriz de cambio de base

(6)#\[\begin{eqnarray} \ket{v} ~= ~ \sum_{i} v_i \ket{e_i} &~=~ \sum_j \tilde v_j \ket{\tilde e_j} \nonumber\\ &~=~ \sum_j \tilde v_j \left( \sum_{i} C_{ij} \ket{e_i}\right) \nonumber \\ &~=~ \sum_{j}\sum_{i} \tilde v_j C_{ij} \ket{e_i} \nonumber \\ &~=~ \sum_i \left( \sum_j C_{ij} \tilde v_j \right) \ket{e_i} \nonumber \\ \end{eqnarray}\]

Comparando obtenemos la relaci√≥n de coeficientes

\[ v_i = \sum_{j} C_{i\color{black}{j}} \tilde v_{\color{black}{j}} \]

Esta expresi√≥n devuelve las componentes \(v_i\) de un vector \(\ket{v}\) en la base \(\{ \ket{e_i}\}\) suponiendo que son conocidas las \(\tilde v_i\) en la base \(\{ \ket{\tilde e_i} \}\)

Nota

  1. La forma en que est√°n sumados los √≠ndices hace que, en notaci√≥n matricial esta operaci√≥n se represente como sigue

\[\begin{split} \begin{pmatrix} v_1\\ \vdots \\ v_n\end{pmatrix} = \begin{pmatrix} C_{11} & \cdots & C_{1n} \\ \vdots & \ddots & \vdots \\ C_{n1} & \cdots & C_{nn} \end{pmatrix} \begin{pmatrix} \tilde v_1\\ \vdots \\ \tilde v_n\end{pmatrix} \end{split}\]

  1. La misma matriz \(C\), que lleva \(\ket{e_i} \to \ket{\tilde e_i}\), lleva \(\tilde v_j \to v_j\)

Un vector es un objeto abstracto, un elemento de un espacio vectorial. La definici√≥n que dimos al principio es f√°cil de entender, pero conlleva la elecci√≥n impl√≠cita de una base. En cada base un vector se representa con un conjunto de componentes diferentes

Ejemplo

Vamos a hacer un test de consistencia con vectores que conocemos en ambas bases. Escojamos por ejemplo \(\ket{v} = \ket{\tilde e_1}\). Las componentes \(\tilde v_i\) de este vector en la base \(\{\ket{\tilde e_i}\}\) son claramente \(\tilde v_i = \begin{pmatrix}1\\0\end{pmatrix}\). Entonces podemos obtener las componentes \(v_i\) en la base \(\{\ket{e_i}\}\) siguiendo la regla encontrada

\[\begin{split} \ket{\tilde e_1} \sim \begin{pmatrix} \tilde v_1\\ \tilde v_2\end{pmatrix} = \begin{pmatrix} 1\\ 0 \end{pmatrix} ~\hbox{en la base} \{\ket{\tilde e_i}\}~~~~\Longrightarrow ~~~~\ket{\tilde e_1} \sim \begin{pmatrix} v_1\\v_2\end{pmatrix} = \begin{pmatrix} 1 & 1 \\ i & -i \end{pmatrix}\begin{pmatrix} 1\\ 0 \end{pmatrix} = \begin{pmatrix} 1\\ i \end{pmatrix}~\hbox{en la base} \{\ket{e_i}\} \end{split}\]

y, an√°logamente, escogiendo \(\ket{v} = \ket{\tilde e_2}\)

\[\begin{split} \ket{\tilde e_2} \sim \begin{pmatrix} \tilde v_1\\ \tilde v_2\end{pmatrix} = \begin{pmatrix} 0\\ 1 \end{pmatrix} ~\hbox{en la base} \{\ket{\tilde e_i}\}~~~~\Longrightarrow ~~~~ \ket{\tilde e_2} \sim \begin{pmatrix} v_1\\v_2\end{pmatrix} = \begin{pmatrix} 1 & 1 \\ i & -i \end{pmatrix}\begin{pmatrix} 0\\ 1 \end{pmatrix} = \begin{pmatrix} 1\\ -i \end{pmatrix} ~\hbox{en la base} \{\ket{e_i}\} \end{split}\]

Si queremos obtener las componentes \(\tilde v_i\) a partir de las \(v_i\), tenemos que invertir la matriz

\[ \tilde v_i = \sum_{j} C^{-1}_{i\color{black}{j}} v_{\color{black}{j}} \]

Ejemplo (continuaci√≥n)

Continuando con el ejemplo anterior, invertir la matriz da

\[\begin{split} C^{-1}_{ij} = \frac{1}{2}\begin{pmatrix} 1 &- i\\ 1 & i \end{pmatrix} \end{split}\]

lo cual nos permite encontrar las componentes de \(\ket{e_1}\) en la base \(\ket{\tilde e_i}\)

\[\begin{split} \ket{e_1} = \ket{v} = \begin{pmatrix} \tilde v_1\\\tilde v_2\end{pmatrix} = \frac{1}{2}\begin{pmatrix} 1 &- i\\ 1 & i \end{pmatrix}\begin{pmatrix} 1\\ 0 \end{pmatrix} = \frac{1}{2}\begin{pmatrix} 1\\ 1 \end{pmatrix}~~~~\hbox{en la base} \{\ket{\tilde e_i}\} \end{split}\]

y tambi√©n

\[\begin{split} \ket{e_2} = \ket{v} = \begin{pmatrix} \tilde v_1\\\tilde v_2\end{pmatrix} = \frac{1}{2}\begin{pmatrix} 1 &- i\\ 1 & i \end{pmatrix}\begin{pmatrix} 0\\ 1 \end{pmatrix} = \frac{1}{2}\begin{pmatrix} -i\\ i \end{pmatrix}~~~~\hbox{en la base} \{\ket{\tilde e_i}\} \end{split}\]

Subespacios Vectoriales#

Seal \(B = \{\ket{e_i}\}_{i=1,...,N}\) la base de un espacio vectorial \(V\). Cualquier subconjunto de elementos de dicha base, \(\bar B\subset B\), genera un subespacio vectorial \(\bar V \subset V\)

Suma Directa

Por ejemplo, podemos dividir \(B\) en dos subconjuntos complementarios

\[B_1 = \{\ket{e_i}\}_{i=1,...,N_1}~~~~~~~,~~~~~~~~~ B_2 = \{\ket{e_i}\}_{i={N_1}+1,...,N}\]

de forma que \(B = B_1 \cup B_2\)

Cada subconjunto ser√° base de un subespacio vectorial \(B_1 \to V_1\) y \(B_2\to V_2\) de forma que el espacio original ser√° la suma directa de ambos

\[ V = V_1 + V_2 \]

Cualquier vector \(\ket{u}\in V\) se escribir√° en la forma \(\ket{v} = \ket{v_1} + \ket{v_2}\) de vectores pertenecientes a cada subespacio.

Por construcci√≥n, la dimensi√≥n de una suma directa es la suma de las dimensiones \(\hbox{dim}\, V = \hbox{dim}\, V_1 + \hbox{dim}\, V_2\)

Espacios de Hilbert#

Conjugaci√≥n adjunta#

La operaci√≥n conjugaci√≥n adjunta, \(\dagger\), es una extensi√≥n de la conjugaci√≥n compleja de \({\mathbb C}\) a todos los elementos de \(\Hil\)

Asociado a cada ket \(\ket{u}\), definimos un vector adjunto, o bra \(\bra{u}\equiv\left(\ket{u}\right)^\dagger\), que representamos mediante un vector fila con las componentes conjugadas complejas.


$\( \dagger \,: \quad\,|u\rangle = \begin{pmatrix} {u_1}\\ {u_2}\\ \vdots \\ {u_N} \end{pmatrix} ~~~~~{\rightarrow}~~~~~~ \left(\ket{u}\right)^\dagger \equiv \bra{u} =\begin{pmatrix} {u_1^*} & {u_2^*} & \cdots & {u_N^*} \end{pmatrix} \)$

Consistentemente encontramos para el producto de un vector por un n√∫mero complejo \(\lambda\)

\[ \dagger \,:\quad\, \lambda\ket{u}=\ket{\lambda u} ~~~~~{\rightarrow}~~~~~~ \left(\lambda\ket{u}\right)^\dagger=\lambda^*\bra{u} = \bra{u}\lambda^* = \bra{\lambda u} \]

ya que el producto de un vector por un n√∫mero es conmutativo.

Al igual que la conjugaci√≥n compleja, la conjugaci√≥n adjunta es una involuci√≥n: su aplicaci√≥n sucesiva devuelve el vector original

\[ (\ket{u}^\dagger)^\dagger =\bra{u}^\dagger = \ket{u} \]

es decir, \(\dagger^2 = I\), el operador identidad.

Definici√≥n: Espacio de Hilbert

Un espacio de Hilbert, \({\Hil}\), es un espacio vectorial dotado de una operaci√≥n interna denominada producto escalar.

Producto Escalar#

Definici√≥n: Producto escalar

El producto escalar de dos vectores \(\ket{u}\) y \(\ket{v}\) es un n√∫mero complejo \(a\in{\mathbb C}\) que denotamos braket

\[ a \equiv \braket{u}{v} \]

si verifica las propiedades siguientes linealidad: \(\to \bra{u}\big(\ket{v}+\ket{w}\big) = \braket{u}{v} + \braket{u}{w}\) hermiticidad: \(\to \braket{v}{u} = \braket{u}{v}^*\) positividad: \(\braket{u}{u} >0\) para todo ket \(\ket{u}\neq 0\) no-degeneraci√≥n: \(~\) si \(\braket{u}{v} = 0\) para todo \(\bra{u}\), entonces necesariamente \(\ket{v}=0\)

Combinando ambas propiedades, el producto escalar tambi√©n es lineal en el primer argumento

\[(\bra{u}+\bra{w})\ket{v} = \braket{u}{v} + \braket{w}{v}\]

Notar que la propiedad de hermiticidad es precisamente la extensi√≥n de la conjugaci√≥n compleja de la que habl√°bamos al definir la aplicaci√≥n adjunta

\[ \braket{u}{v}^* = \braket{u}{v}^\dagger = \braket{v}{u} \]

Hemos usado que \(\bra{u}^\dagger = \ket{u}\) y \(\ket{v}^\dagger = \bra{v}\) pero le hemos a√±adido una regla m√°s: al tomar el mapa adjunto es necesario invertir el orden de los elementos

De no haber seguido esta regla, habr√≠amos obtenido un resultado err√≥neo

\[ \braket{u}{v}^\dagger \to \ket{u}\bra{v} \]

que no es ni siguiera un n√∫mero complejo.

Norma#

Una norma es un una funci√≥n real \(\|\cdot\| : \Lin(\Hil) \to {\mathbb R}\) con las siguientes propiedades

  • ser definida positiva \(\|A\|\geq 0\) con \(\| A\| = 0 \Leftrightarrow A= 0\)

  • homogeneidad \(\|\lambda A\| = |\lambda| \|A\|\)

  • triangle inequality. \(\|A+B\| \leq \| A\| + \|B\|\)

Un espacio de Hilbert es, autom√°ticamente un espacio normado. La positividad del producto escalar de un vector por s√≠ mismo permite definir su norma.

\[ \|\ket{v}\| = \sqrt{\braket{v}{v}} \]

Nota

  • En contraste con la definici√≥n de producto escalar en espacios vectoriales reales, en el caso complejo se hace necesario conjugar el bra, para que la norma de un vector sea siempre real y positiva. Esta es la idea detr√°s de la definici√≥n de la conjugaci√≥n adjunta.

  • El √∫nico vector que tiene norma nula en un espacio de Hilbert es el elemento neutro

\[ \braket{v}{v} = 0 ~~~¬†\Leftrightarrow ~~~\ket{v} = 0 \]

Distancia#

Dados dos elementos \(\ket{u}\) y \(\ket{v}\) de \(\Hil\), podemos definir la distancia entre ellos como la norma de su diferencia

\[ d(\rule{0mm}{5mm}\ket{v},\ket{w}) = \| \ket{v}-\ket{w}\| \]

En particular

  • \(d(\ket{v}, \ket{w}) = d(\ket{w}, \ket{v})\)

  • \(d(\ket{v}, \ket{v}) = 0\)

Bases ortonormales#

Hasta ahora, a los vectores de una base \(\{\ket{e_i}\}\) s√≥lo se les ha pedido que sean \(N\) vectores linealmente independientes, donde \(N\) es la dimensi√≥n del espacio vectorial \(\V\):

En un espacio de Hilbert \(\Hil\) tiene sentido calcular el producto escalar de dos elementos de una base.

Definici√≥n: Base ortonormal

Una base ortornormal se caracteriza por la siguiente lista de productos escalares

\[ \braket{e_i}{e_j} = \delta_{ij} \]

  • Por un lado, dos elementos distintos de la base son ortogonales \(\braket{e_1}{e_2} = 0\).

  • Por otro, todos est√°n normalizados \( \| e_i \| = \sqrt{\braket{e_1}{e_1}} = \sqrt{1} = 1\).

Teorema: de Gram-Schmidt

Dada una base general \(\{\braket{f_i}{f_j}\neq \delta_{ij}\}\) de vectores no ortonormales, existe una procedimiento iterativo (de Gram-Schmidt ) para construir, a partir de ella, una nueva base ortonormal \(\{\braket{e_i}{e_j}\}=\delta_{ij}\)

Dado un vector \(\ket{v} = \sum_{i=1}^N v_i \ket{e_i}\) escrito en una base ortonormal, la componente \(v_i\) se extrae mediante la proyecci√≥n ortogonal

$\( v_i =\braket{e_i}{v} \)$
